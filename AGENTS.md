# Overview

This file provides guidance for coding agents when working with code in this repository.

## Project Summary

Tip Genius is an AI-powered soccer match prediction system that combines Large Language Models (LLMs) with real-world odds data to generate informed predictions. The system features a modern Progressive Web App (PWA) frontend and a Python-based prediction engine that runs on Vercel's serverless platform.

### Generated Artifacts & Large Directories

- **Edit source files, not generated outputs**:
  - Edit `src/css/styles.css`, not `public/css/tailwind.css`
  - Treat `public/version.json` as generated by `npm run generate:version` or `npm run build`
- **Avoid scanning or modifying large historical artifact directories unless the task explicitly targets them**:
  - `data/`
  - `src/tip_genius/logs/`
  - `public/images/teams_bak/`
  - files and folders matching `*_bak*`
- **Primary source-of-truth paths**:
  - `src/tip_genius/` for Python runtime code
  - `src/tip_genius/cfg/` for YAML-driven provider, API, and app configuration
  - `public/` for shipped frontend assets and UI code, excluding generated files noted above

## Architecture Overview

### Core Components

- **Frontend (`public/`)**: Static PWA with Vanilla JavaScript, Tailwind CSS, and Service Workers
- **Python Engine (`src/tip_genius/`)**: Main prediction logic and LLM integration
- **Configuration (`src/tip_genius/cfg/`)**: YAML configs for APIs, LLMs, and application settings
- **Data Processing (`src/tip_genius/lib/`)**: Modular libraries for API handling, LLM management, storage, and team matching

### Key Python Modules

- `tip_genius.py`: Main workflow orchestrator and TipGenius class
- `lib/llm_manager.py`: Handles multiple LLM providers (Mistral, OpenAI, Google Gemini, etc.)
- `lib/api_data.py`: Fetches and processes odds data from external APIs
- `lib/storage_manager.py`: Manages data persistence to Vercel KV and file system
- `lib/team_matching.py`: Fuzzy matching for team logos and names

### LLM Integration Strategy

The system uses a lightweight approach with direct API calls instead of heavy LLM libraries:

- Supports multiple providers: Mistral, Google Gemini, DeepSeek Chat, OpenAI ChatGPT, Meta Llama, Microsoft Phi, Anthropic Claude, Grok, and more (via OpenRouter)
- Uses `requests` library for HTTP calls rather than provider-specific SDKs
- Implements rate limiting and retry logic with exponential backoff
- Maintains minimal dependencies for better performance and deployment speed

### Data Flow

1. **Odds Fetching**: External API provides real-time soccer odds
2. **Processing**: Data normalized and enriched with team information
3. **LLM Prediction**: Multiple LLM providers generate predictions with reasoning
4. **Validation**: Predictions validated for consistency with odds
5. **Storage**: Results stored in Vercel KV (Redis) and optionally as files
6. **Frontend**: PWA displays predictions with team logos and dark mode support

### LLM Provider Changes

When adding or changing an LLM provider, agents should review the full integration path instead of editing a single file:

1. `src/tip_genius/cfg/llm_config.yaml` for provider API/model/request configuration
2. `src/tip_genius/cfg/tip_genius_config.yaml` to enable or disable the provider in runtime combinations
3. `.github/workflows/match-predictions.yaml` if a new secret or environment variable is required
4. `.env.example` and user-facing docs if setup steps or required keys change
5. Frontend assets such as `public/images/llm-logos/` and UI code if the provider is exposed in the web app

OpenRouter headers are currently configured in `src/tip_genius/cfg/llm_config.yaml` via `optional_headers`.

## Dependency Pinning & Compatibility

### Tailwind CSS v3 (Pinned)

**Status**: Pinned to v3.4.x - Major version upgrades blocked by Dependabot

**Reason**: Tailwind CSS v4 introduces breaking changes requiring significant configuration updates:

- CSS `@source` directive replaces `content` config option
- Config format changes and removal of JavaScript-based content paths
- Requires migration of `tailwind.config.js` to new syntax

**Configuration**:

- **package.json**: `"tailwindcss": "^3.4"` (allows patch updates, blocks v4+)
- **`.github/dependabot.yml`**: Explicit ignore rule for v4.0.0 and above

**Migration Path** (when needed):

1. Update `tailwind.config.js` to use `@source` in CSS
2. Move content globs from JavaScript to CSS directives
3. Update CSS import statements in `src/css/styles.css`
4. Update build command if necessary
5. Test all styling before deploying

**Reference**: [Tailwind CSS v4 Migration Guide](https://tailwindcss.com/docs/upgrade-guide)

- All prediction data is timestamped and can include debug suffixes

- OpenRouter integration uses optional headers defined in `src/tip_genius/cfg/llm_config.yaml`
