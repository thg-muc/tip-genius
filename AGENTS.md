# Overview

This file provides guidance for coding agents when working with code in this repository.

## Project Summary

Tip Genius is an AI-powered soccer match prediction system that combines Large Language Models (LLMs) with real-world odds data to generate informed predictions. The system features a modern Progressive Web App (PWA) frontend and a Python-based prediction engine that runs on Vercel's serverless platform.

## Development Commands

```bash
# Install dependencies
uv sync && uv sync --group dev && npm install

# Build frontend assets
npm run build

# Watch CSS during development
npm run watch:css

# Verification commands (match CI)
uv run ruff check src/
uv run ruff format --check src/
uv run pyright src/
npx prettier --check "public/**/*.{js,css,json}"

# Autofix
uv run ruff check --fix src/ && uv run ruff format src/
npx prettier --write "public/**/*.{js,css,json}" "api/**/*.js" "src/**/*.{yaml,yml}" ".github/**/*.yaml" "utils/**/*.py"

# Pre-commit hooks (run once after cloning)
uv run pre-commit install
uv run pre-commit install --hook-type commit-msg

# Run prediction workflow locally with debug logging
cd src/tip_genius && DEBUG_MODE=TRUE uv run python tip_genius.py
# Logs written to src/tip_genius/logs/ — load .env.local from repo root

# Smoke tests (live LLM calls — costs real money, run on demand only)
RUN_SMOKE=1 uv run pytest tests/smoke -m smoke -v
```

## Smoke Tests

`tests/smoke/test_tip_genius_workflow.py` invokes every active provider via
`LLMManager.get_prediction()`. Gated by the `smoke` marker and `RUN_SMOKE=1`
(deselected by default). Each run costs real API spend — never execute
automatically. **Suggest running** after changes to `cfg/llm_config.yaml`,
`cfg/tip_genius_config.yaml`, or `lib/llm_manager.py`.

## Generated Artifacts & Large Directories

- **Edit source files, not generated outputs**:
  - Edit `src/css/styles.css`, not `public/css/tailwind.css`
  - Treat `public/version.json` as generated by `npm run generate:version` or `npm run build`
- **Avoid scanning or modifying** unless the task explicitly targets them:
  - `data/`, `src/tip_genius/logs/`, `public/images/teams_bak/`, `*_bak*`
- **Primary source-of-truth paths**:
  - `src/tip_genius/` for Python runtime code
  - `src/tip_genius/cfg/` for YAML-driven provider, API, and app configuration
  - `public/` for shipped frontend assets and UI code

## Architecture Overview

### Key Python Modules

- `tip_genius.py`: Main workflow orchestrator and TipGenius class
- `lib/llm_manager.py`: Handles multiple LLM providers (Mistral, OpenAI, Google Gemini, etc.)
- `lib/api_data.py`: Fetches and processes odds data from external APIs
- `lib/storage_manager.py`: Manages data persistence to Vercel KV and file system
- `lib/team_matching.py`: Fuzzy matching for team logos and names

### LLM Provider Changes

When adding or changing an LLM provider, review the full integration path:

1. `src/tip_genius/cfg/llm_config.yaml` for provider API/model/request configuration
2. `src/tip_genius/cfg/tip_genius_config.yaml` to enable or disable the provider in runtime combinations
3. `.github/workflows/match-predictions.yaml` if a new secret or environment variable is required
4. `.env.example` and user-facing docs if setup steps or required keys change
5. Frontend assets such as `public/images/llm-logos/` and UI code if the provider is exposed in the web app

OpenRouter headers are currently configured in `src/tip_genius/cfg/llm_config.yaml` via `optional_headers`.

## Dependency Pinning

### Tailwind CSS v3 (Pinned)

Pinned to `^3.4` — v4 introduces breaking changes (new `@source` directive, config format changes). Dependabot ignore rule in place. See [Tailwind CSS v4 Migration Guide](https://tailwindcss.com/docs/upgrade-guide) when upgrading.

## Development Notes

- Always prefer standard Tailwind classes over arbitrary values (e.g. `w-56` not `w-[224px]`) — arbitrary values may be purged
- **Reasoning Model Support**: OpenRouter reasoning models (GPT-OSS 120b, Grok) use `reasoning.effort` parameter; Google Gemma uses native `thinkingConfig` with `thinkingLevel: minimal` option; Google response parser skips thought parts to extract the actual answer
